BlueRetro's button mapping system translates inputs from a Bluetooth controller into outputs understood by the wired console. Each wired port holds up to **128 mapping entries** (`ADAPTER_MAPPING_MAX = 128`).

## Mapping entry structure

Every entry in the `map_cfg` array describes a single source-to-destination translation.

| Field | Type | Description |
|-------|------|-------------|
| `src_btn` | `uint8_t` | Source button or axis ID from the Bluetooth controller |
| `dst_btn` | `uint8_t` | Destination button or axis ID for the wired console |
| `dst_id` | `uint8_t` | Destination device (wired port index) |
| `perc_max` | `uint8_t` | Maximum output as a percentage (default: `100`) |
| `perc_threshold` | `uint8_t` | Threshold percentage for digital triggers (default: `50`) |
| `perc_deadzone` | `uint8_t` | Deadzone size as a raw scaled value (default: `135`; values above 100 are used for large inner deadzones) |
| `turbo` | `uint8_t` | Turbo mode â€” `0` = off, non-zero = turbo rate |
| `algo` | `uint8_t` | Scaling algorithm to apply |

<Note>
  The default combo mappings added at config initialization use `perc_max = 100`, `perc_threshold = 50`, and `perc_deadzone = 135`.
</Note>

## Device modes

The `dev_mode` field in `out_cfg` controls what kind of device each wired port emulates. This affects which destination button IDs are valid for that port.

| Value | Constant | Description |
|-------|----------|-------------|
| `0` | `DEV_PAD` | Standard gamepad â€” default for all ports |
| `1` | `DEV_PAD_ALT` | Alternate gamepad layout (console-specific; e.g. 6-button Genesis pad) |
| `2` | `DEV_KB` | Keyboard emulation |
| `3` | `DEV_MOUSE` | Mouse emulation |

## Turbo mode

Setting a non-zero value in the `turbo` field of a mapping entry enables turbo (auto-fire) for that button. The value controls the rate: higher values produce faster oscillation. Turbo works for both digital buttons and analog axes mapped to digital outputs.

## Mapping quirks

BlueRetro automatically adjusts mappings for certain controllers to correct known hardware or firmware differences. These are applied transparently via BT flags and do not require manual configuration.
