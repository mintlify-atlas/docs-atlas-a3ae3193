## Overview

BlueRetro runs on the standard ESP32 (dual-core Xtensa LX6 with integrated Bluetooth Classic and BLE). It does **not** support ESP32-S2, ESP32-S3, or ESP32-C3 variants, which lack Bluetooth Classic (BR/EDR) support.

The firmware is built and tested against the ESP32 with a minimum revision of Rev 1. The `sdkconfig` targets a 4 MB flash, 240 MHz CPU, and dual-mode Bluetooth (BR/EDR + BLE) with up to 7 simultaneous connections.

## HW1 vs HW2

BlueRetro hardware comes in two form factors:

<Tabs>
  <Tab title="HW1 — External adapter">
    HW1 is an **external** adapter design intended for breadboards, perfboards, or off-the-shelf ESP32 development boards such as the **ESP32-DevKitC**. It connects to the console's controller port via wires or a purpose-built cable, and the ESP32 module sits outside the console.

    **Best for:**
    - Quick prototyping and testing
    - Users who want a non-invasive, reversible installation
    - N64, SNES, Genesis, and other systems with accessible external controller ports

    **Typical components:**
    - ESP32-DevKitC (38-pin or 30-pin)
    - Breadboard or custom PCB
    - Console-specific controller port connector or cable
  </Tab>
  <Tab title="HW2 — Internal PCB">
    HW2 is an **internal** design where a compact PCB is installed inside the console. It replaces the original controller port PCB or mounts directly to the console's mainboard. This gives a clean, permanent installation with no external cables.

    **Best for:**
    - Permanent installations
    - Users comfortable with soldering
    - Consoles where the controller port circuitry is easily accessed (GameCube, N64, PlayStation)

    **Typical components:**
    - Custom HW2 PCB (community-designed, per-system)
    - Soldering to console controller port pads
  </Tab>
</Tabs>

### Comparison

| Feature | HW1 (External) | HW2 (Internal) |
|---|---|---|
| Form factor | External, uses DevKitC | Internal PCB |
| Installation difficulty | Easy — no soldering required | Moderate — requires soldering |
| Reversibility | Fully reversible | Semi-permanent |
| Appearance | External dongle | Clean internal install |
| Power source | USB or console port | Console internal power |
| Recommended for | Prototyping, multi-system | Single-system permanent build |

## Per-system wiring configurations

The `configs/` directory in the repository contains per-system wiring and build configurations for both HW1 and HW2. Each subdirectory corresponds to a supported target system:

| Config directory | Target system |
|---|---|
| `3do` | 3DO Interactive Multiplayer |
| `cdi` | Philips CD-i |
| `dreamcast` | Sega Dreamcast |
| `gamecube` | Nintendo GameCube |
| `gbahd` | Game Boy Advance HD |
| `genesis` | Sega Mega Drive / Genesis |
| `jaguar` | Atari Jaguar |
| `jvs_arcade` | JVS Arcade |
| `n64` | Nintendo 64 |
| `nes` | Nintendo Entertainment System |
| `parallel_1p` | Parallel 1P (NeoGeo, Supergun, JAMMA, etc.) |
| `parallel_1p_3v3` | Parallel 1P — 3.3 V variant |
| `parallel_2p` | Parallel 2P (Atari 2600/7800, Master System, etc.) |
| `parallel_2p_3v3` | Parallel 2P — 3.3 V variant |
| `pc_engine` | PC Engine / TurboGrafx-16 |
| `pcfx` | NEC PC-FX |
| `playstation` | PlayStation 1 / PlayStation 2 |
| `saturn` | Sega Saturn |
| `snes` | Super Nintendo Entertainment System |
| `universal` | Universal / multi-system |
| `virtualboy` | Nintendo Virtual Boy |
| `wii_extension` | Wii Extension port |

Each config directory contains system-specific `sdkconfig` fragments and GPIO mapping files used at build time. Refer to the [GitHub wiki](https://github.com/darthcloud/BlueRetro/wiki) for pinout diagrams for each system.

<Info>
  The `universal` config is a good starting point if you want to test BlueRetro with multiple systems before committing to a system-specific build.
</Info>

## ESP32 board recommendations

Any standard ESP32 module with at least 4 MB flash should work. The boards most commonly used in the community are:

- **ESP32-DevKitC v4** (38-pin) — recommended for HW1 builds
- **ESP32-DevKitC v4** (30-pin) — also compatible with HW1
- **ESP32-WROOM-32** module — base module used in most community HW2 PCBs

<Warning>
  Do not use ESP32-S2, ESP32-S3, or ESP32-C3 boards. These variants do not support Bluetooth Classic (BR/EDR), which is required for PS3, Wii, and many other controllers.
</Warning>
