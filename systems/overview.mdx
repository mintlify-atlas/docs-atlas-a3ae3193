BlueRetro maps Bluetooth controllers to the native wired protocols of retro game consoles. Each supported system has a numeric ID defined in the `wired system ID` enum in `adapter.h`. The system is selected either manually in the configuration or automatically via `WIRED_AUTO`.

## Hardware configurations

Both HW1 and HW2 variants of BlueRetro support the same set of systems. Configurations are stored under `configs/hw1/` and `configs/hw2/`:

`3do`, `cdi`, `dreamcast`, `gamecube`, `gbahd`, `genesis`, `jaguar`, `jvs_arcade`, `n64`, `nes`, `parallel_1p`, `parallel_1p_3v3`, `parallel_2p`, `parallel_2p_3v3`, `pc_engine`, `pcfx`, `playstation`, `saturn`, `snes`, `universal`, `virtualboy`, `wii_extension`

## Multitap modes

Multitap expansion is controlled by the `multitap_cfg` field in `global_cfg`. The available modes are defined in `adapter.h`:

| Mode | Description |
|---|---|
| `MT_NONE` | No multitap; standard 1â€“2 player mode |
| `MT_SLOT_1` | Multitap connected in port 1 |
| `MT_SLOT_2` | Multitap connected in port 2 |
| `MT_DUAL` | Multitap connected in both ports |
| `MT_ALT` | Alternate multitap protocol (e.g., EA 4-Way Play for Genesis) |

## Accessory modes

Each output port's `acc_mode` field controls which peripheral is emulated in the controller pak slot:

| Mode | Description |
|---|---|
| `ACC_NONE` | No accessory |
| `ACC_MEM` | Memory card / controller pak |
| `ACC_RUMBLE` | Rumble pak |
| `ACC_BOTH` | Both memory and rumble (where supported) |

## Device modes

Each output port's `dev_mode` controls the type of input device presented to the console:

| Mode | Description |
|---|---|
| `DEV_PAD` | Standard gamepad |
| `DEV_PAD_ALT` | Alternate gamepad layout (e.g., 6-button, 3D pad) |
| `DEV_KB` | Keyboard |
| `DEV_MOUSE` | Mouse |
